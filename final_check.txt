[23] 이렇게 하면 한 곡이 다차원 공간의 한 점으로 표현된다. 예를 들어 잔잔한 발라드 한 곡은 (에너지=0.2, 밝기=0.3, 어쿠스틱=0.9, 댄서빌리티=0.3) 이런 식이다.
[24] 곡 = (x₁, x₂, ..., xₙ)
[25] 두 곡이 얼마나 비슷한지는 두 벡터 사이의 유클리드 거리로 나타낼 수 있다. 중학교 때 배운 피타고라스 정리를 n차원으로 확장한 것이다.
[26] d(A, B) = √[(a₁-b₁)² + (a₂-b₂)² + ... + (aₙ-bₙ)²]
[27] 거리가 작을수록 두 곡의 음악적 특징이 비슷하다.
[28] 참고로 비슷한 용도로 코사인 유사도도 쓰이는데, 이건 두 벡터의 방향만 본다. 그런데 음악에서는 방향보다 실제 수치 차이가 더 중요하다. 템포 60짜리 곡과 140짜리 곡은 방향이 비슷해 보여도 느낌이 완전히 다르기 때문에 유클리드 거리가 더 적합하다.

[37] 5. 경사하강법과 미분계수
[38] 머신러닝 모델이 학습한다는 게 어떤 의미인지 이해하려면 경사하강법을 알아야 한다. 경사하강법은 손실 함수 L(w)를 최소화하는 w를 찾아가는 방법이다. 손실 함수는 모델의 예측이 실제 정답과 얼마나 틀렸는지를 수치로 나타낸 함수고, w는 모델이 조정하는 가중치다. 가장
[39] L(w) = (1/m) × Σ (예측값 - 실제값)²
[40] 이걸 이해하려면 먼저 미분계수부터 짚어야 한다. 함수 f(x)에서 x = a일 때의 평균변화율은 다음과 같다.
[41] [f(a+h) - f(a)] / h  (h ≠ 0)
[42] 이건 두 점을 이은 직선, 즉 할선의 기울기다. h를 점점 0에 가깝게 줄이면 할선이 점점 접선에 가까워지고, 그 극한값이 x = a에서의 미분계수다.
[43] f'(a) = lim(h→0) [f(a+h) - f(a)] / h
[44] 미분계수는 그 점에서 함수가 얼마나 빠르게, 어느 방향으로 변하는지를 알려준다. 기울기가 양수면 x가 커질수록 f(x)도 커지고, 음수면 줄어든다.
[45] 경사하강법은 바로 이 미분계수를 이용한다. 현재 위치에서 기울기(미분값)의 반대 방향으로 조금씩 이동하면 결국 최솟값에 도달한다는 아이디어다. 산에서 경사를 따라 내려가는 것과 같아서 경사하강법이라는 이름이 붙었다.
[46] w_new = w_old - α × dL/dw
[47] 여기서 α는 학습률로 한 번에 얼마나 이동할지를 정하는 상수다. 이 과정을 반복하면 w가 손실 함수의 최솟값에 해당하는 지점으로 수렴한다.
[48] L(w) = (w-3)² 라 하면 w = 3일 때 최솟값 0이다. 미분하면 dL/dw = 2(w-3)이고, w₀ = 0, α = 0.3으로 시작해보자.
[50] 반복할수록 w가 3에 수렴하고 L(w)도 0에 가까워진다. 미분계수를 이용해서 최솟값 방향으로 조금씩 이동하는 것, 이게 경사하강법의 전부다.
[51] 실제 머신러닝에서는 가중치 w가 수백만 개인데, 이럴 때는 변수마다 편미분해서 기울기 벡터(gradient)를 만든다.
[52] ∇L = (∂L/∂w₁, ∂L/∂w₂, ..., ∂L/∂wₙ)
[53] w_new = w_old - α × ∇L(w_old)
[54] 딥러닝도 결국 이 공식을 반복하는 것이다. 복잡해 보이는 신경망 안에서도 핵심은 미분계수를 이용한 경사하강이다.
[55] α가 너무 크면 최솟값을 지나쳐서 발산할 수 있고, 너무 작으면 수렴이 너무 느려진다. 적절한 값을 찾는 것도 머신러닝 실무에서 중요한 작업이다.
[56] 6. 취향 확장 시스템 설계
[57] 4장의 문제의식을 바탕으로 필터버블을 수학적으로 극복하는 방법을 설계해봤다.
[58] 첫 번째는 거리 밴드 방식이다. 기준 곡에서 거리가 최솟값도 최댓값도 아닌 특정 구간 안에 들어오는 곡만 후보로 선정한다.
[59] S = { B | d_min ≤ d(A, B) ≤ d_max }
[60] 너무 가까우면 뻔하고 너무 멀면 거부감이 든다는 직관을 수식으로 나타낸 것이다.
[61] 두 번째는 가중 유클리드 거리다. 특징마다 가중치를 다르게 준다.
[62] d_w(A, B) = √[w₁(a₁-b₁)² + w₂(a₂-b₂)² + ... + wₙ(aₙ-bₙ)²]
[63] 핵심은 감정과 관련된 특징인 밝기, 에너지에는 가중치를 크게 줘서 감정선은 비슷하게 유지하고, 장르나 악기 관련 특징은 가중치를 작게 줘서 스타일은 자유롭게 두는 것이다.
[64] w_밝기 = 2.0,  w_에너지 = 1.5,  w_어쿠스틱 = 1.0,  w_댄서빌리티 = 0.8
[65] 그러면 감정선은 비슷한데 장르는 완전히 다른 세렌디피티 있는 추천이 가능해진다.
[66] 이 가중치 값들도 경사하강법으로 학습할 수 있다. 사용자가 추천곡을 끝까지 들으면 손실을 줄이고, 건너뛰면 손실을 늘리는 식으로 피드백을 설계한 다음, 기울기 방향으로 w를 조정하면 된다. 경사하강법이 음악 추천 시스템과 연결되는 지점이 여기다.
