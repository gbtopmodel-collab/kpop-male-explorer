[5][Normal] 함수의 극솟값과 유클리드 거리를 이용한 음악 취향 확장 모델 설계
[6][Normal] 미분계수와 경사하강법을 중심으로
[15][Normal] 미적분 주제탐구 보고서
[17][Normal] 1. 서론
[18][Normal] 음악 앱을 오래 쓰다 보면 어느 순간부터 추천 목록이 항상 비슷한 노래들로만 채워진다는 걸 느끼게 된다. 처음엔 취향이 그런가 싶었는데, 알고 보니 이게 알고리즘 때문이었다. 추천 시스템이 내가 좋아한 것들을 기반으로 계속 비슷한 것들만 보여주다 보니, 새로운 음악을 접할 기회가 점점 줄어드는 것이다. 이런 현상을 필터버블(filter bubble)이라고 
[19][Normal] 그렇다면 추천 알고리즘은 어떻게 작동하는 걸까. 이걸 수학적으로 이해해보고 싶었다. 특히 머신러닝에서 핵심적으로 쓰이는 경사하강법이라는 개념이 고등학교에서 배우는 미분과 어떻게 연결되는지 탐구해보는 것이 이 보고서의 목표다.
[20][Normal] 2. 노래를 수학으로 표현하기
[21][Normal] 컴퓨터가 음악의 유사도를 계산하려면 먼저 음악을 숫자로 바꿔야 한다. 이렇게 한 곡의 특징들을 숫자로 나열한 것을 특징 벡터(feature vector)라고 한다. 예를 들면 에너지, 밝기(valence), 어쿠스틱 정도, 댄서빌리티, 템포 같은 항목들이다.
[23][Normal] 이렇게 하면 한 곡이 다차원 공간의 한 점으로 표현된다. 예를 들어 잔잔한 발라드 한 곡은 (에너지=0.2, 밝기=0.3, 어쿠스틱=0.9, 댄서빌리티=0.3) 이런 식이다.
[24][Normal] 곡 = (x₁, x₂, ..., xₙ)
[25][Normal] 두 곡이 얼마나 비슷한지는 두 벡터 사이의 유클리드 거리로 나타낼 수 있다. 중학교 때 배운 피타고라스 정리를 n차원으로 확장한 것이다.
[26][Normal] d(A, B) = √[(a₁-b₁)² + (a₂-b₂)² + ... + (aₙ-bₙ)²]
[27][Normal] 거리가 작을수록 두 곡의 음악적 특징이 비슷하다.
[28][Normal] 참고로 비슷한 용도로 코사인 유사도도 쓰이는데, 이건 두 벡터의 방향만 본다. 그런데 음악에서는 방향보다 실제 수치 차이가 더 중요하다. 템포 60짜리 곡과 140짜리 곡은 방향이 비슷해 보여도 느낌이 완전히 다르기 때문에 유클리드 거리가 더 적합하다.
[29][Normal] 또 하나 중요한 게 정규화다. 템포는 0~250 범위고 다른 특징들은 0~1인데, 이 상태에서 거리를 계산하면 템포가 거리를 거의 독점한다. 두 곡 템포 차이가 80이면 그것만으로 거리에 6400이 더해지는데, 에너지 차이 0.5는 고작 0.25밖에 안 된다. 그래서 모든 특징을 0~1 범위로 맞추는 정규화가 먼저 필요하다.
[30][Normal] x' = (x - x_min) / (x_max - x_min)
[31][Normal] 3. 애플 뮤직은 실제로 어떻게 작동할까
[32][Normal] 애플 뮤직의 정확한 알고리즘은 공개된 게 없다. 다만 알려진 바로는 크게 세 방식을 섞어서 쓴다. 첫 번째는 곡의 음향 특징을 분석해서 비슷한 곡을 추천하는 콘텐츠 기반 필터링, 두 번째는 나와 비슷한 취향을 가진 사람들이 들은 곡을 추천하는 협업 필터링, 세 번째는 음악 전문가가 직접 고른 플레이리스트인 편집자 큐레이션이다. 여기에 재생 완료, 좋아요, 
[33][Normal] 이 보고서에서 수학적으로 분석하는 건 그중 콘텐츠 기반 필터링 부분이다. 특징 벡터와 유클리드 거리가 직접적으로 사용되는 게 여기이기 때문이다.
[34][Normal] 4. 가장 먼 곡이 좋은 추천일까
[35][Normal] 필터버블을 벗어나려면 기준 곡과 거리가 먼 곡을 추천하면 되지 않을까 싶었다. 그런데 조금만 생각해보면 문제가 있다. 기준 곡에서 거리가 가장 먼 곡은 특징이 정반대인 곡이다. 잔잔한 발라드를 즐겨 듣는 사람한테 거리 최대인 곡을 추천하면 빠르고 시끄럽고 어두운 익스트림 메탈이 나올 수 있다. 이건 새로운 발견이 아니라 그냥 불쾌한 경험이다.
[36][Normal] 추천 시스템 연구에서는 좋은 추천의 기준을 참신성(아직 들어본 적 없는 곡), 다양성(추천 목록이 서로 다양하게 분포), 세렌디피티(의외인데 마음에 드는 발견) 이렇게 세 가지로 나눈다. 진짜 목표는 세 번째인 세렌디피티다. 이걸 수학적으로 표현하면 기준 곡에서 너무 가깝지도 너무 멀지도 않은, 딱 적당한 거리 안에 있는 곡을 찾는 것이다.
[37][Normal] 5. 경사하강법과 미분계수
[38][Normal] 머신러닝 모델이 학습한다는 게 어떤 의미인지 이해하려면 경사하강법을 알아야 한다. 경사하강법은 손실 함수 L(w)를 최소화하는 w를 찾아가는 방법이다. 손실 함수는 모델의 예측이 실제 정답과 얼마나 틀렸는지를 수치로 나타낸 함수고, w는 모델이 조정하는 가중치다. 가장 흔하게 쓰는 게 평균 제곱 오차(MSE)다.
[39][Normal] L(w) = (1/m) × Σ (예측값 - 실제값)²
[40][Normal] 이걸 이해하려면 먼저 미분계수부터 짚어야 한다. 함수 f(x)에서 x = a일 때의 평균변화율은 다음과 같다.
[41][Normal] [f(a+h) - f(a)] / h  (h ≠ 0)
[42][Normal] 이건 두 점을 이은 직선, 즉 할선의 기울기다. h를 점점 0에 가깝게 줄이면 할선이 점점 접선에 가까워지고, 그 극한값이 x = a에서의 미분계수다.
[43][Normal] f'(a) = lim(h→0) [f(a+h) - f(a)] / h
[44][Normal] 미분계수는 그 점에서 함수가 얼마나 빠르게, 어느 방향으로 변하는지를 알려준다. 기울기가 양수면 x가 커질수록 f(x)도 커지고, 음수면 줄어든다.
[45][Normal] 경사하강법은 바로 이 미분계수를 이용한다. 현재 위치에서 기울기(미분값)의 반대 방향으로 조금씩 이동하면 결국 최솟값에 도달한다는 아이디어다. 산에서 경사를 따라 내려가는 것과 같아서 경사하강법이라는 이름이 붙었다.
[46][Normal] w_new = w_old - α × dL/dw
[47][Normal] 여기서 α는 학습률로 한 번에 얼마나 이동할지를 정하는 상수다. 이 과정을 반복하면 w가 손실 함수의 최솟값에 해당하는 지점으로 수렴한다.
[48][Normal] L(w) = (w-3)² 라 하면 w = 3일 때 최솟값 0이다. 미분하면 dL/dw = 2(w-3)이고, w₀ = 0, α = 0.3으로 시작해보자.
[50][Normal] 반복할수록 w가 3에 수렴하고 L(w)도 0에 가까워진다. 미분계수를 이용해서 최솟값 방향으로 조금씩 이동하는 것, 이게 경사하강법의 전부다.
[51][Normal] 실제 머신러닝에서는 가중치 w가 수백만 개인데, 이럴 때는 변수마다 편미분해서 기울기 벡터(gradient)를 만든다.
[52][Normal] ∇L = (∂L/∂w₁, ∂L/∂w₂, ..., ∂L/∂wₙ)
[53][Normal] w_new = w_old - α × ∇L(w_old)
[54][Normal] 딥러닝도 결국 이 공식을 반복하는 것이다. 복잡해 보이는 신경망 안에서도 핵심은 미분계수를 이용한 경사하강이다.
[55][Normal] α가 너무 크면 최솟값을 지나쳐서 발산할 수 있고, 너무 작으면 수렴이 너무 느려진다. 적절한 값을 찾는 것도 머신러닝 실무에서 중요한 작업이다.
[56][Normal] 6. 취향 확장 시스템 설계
[57][Normal] 4장의 문제의식을 바탕으로 필터버블을 수학적으로 극복하는 방법을 설계해봤다.
[58][Normal] 첫 번째는 거리 밴드 방식이다. 기준 곡에서 거리가 최솟값도 최댓값도 아닌 특정 구간 안에 들어오는 곡만 후보로 선정한다.
[59][Normal] S = { B | d_min ≤ d(A, B) ≤ d_max }
[60][Normal] 너무 가까우면 뻔하고 너무 멀면 거부감이 든다는 직관을 수식으로 나타낸 것이다.
[61][Normal] 두 번째는 가중 유클리드 거리다. 특징마다 가중치를 다르게 준다.
[62][Normal] d_w(A, B) = √[w₁(a₁-b₁)² + w₂(a₂-b₂)² + ... + wₙ(aₙ-bₙ)²]
[63][Normal] 핵심은 감정과 관련된 특징인 밝기, 에너지에는 가중치를 크게 줘서 감정선은 비슷하게 유지하고, 장르나 악기 관련 특징은 가중치를 작게 줘서 스타일은 자유롭게 두는 것이다.
[64][Normal] w_밝기 = 2.0,  w_에너지 = 1.5,  w_어쿠스틱 = 1.0,  w_댄서빌리티 = 0.8
[65][Normal] 그러면 감정선은 비슷한데 장르는 완전히 다른 세렌디피티 있는 추천이 가능해진다.
[66][Normal] 이 가중치 값들도 경사하강법으로 학습할 수 있다. 사용자가 추천곡을 끝까지 들으면 손실을 줄이고, 건너뛰면 손실을 늘리는 식으로 피드백을 설계한 다음, 기울기 방향으로 w를 조정하면 된다. 경사하강법이 음악 추천 시스템과 연결되는 지점이 여기다.
[67][Normal] 7. 직접 계산해보기
[68][Normal] 정규화된 5곡을 다음과 같이 설정했다.
[70][Normal] 기준 곡 A에서의 유클리드 거리를 계산하면 다음과 같다.
[71][Normal] d(A,B) = √[(0.05)²+(0.05)²+(0.05)²+(0.05)²] = √0.01 ≈ 0.10
[72][Normal] d(A,E) = √[(0.10)²+(0.30)²+(0.10)²+(0.20)²] = √0.15 ≈ 0.39
[73][Normal] d(A,D) = √[(0.75)²+(0.10)²+(0.85)²+(0.10)²] = √1.305 ≈ 1.14
[74][Normal] d(A,C) = √[(0.70)²+(0.50)²+(0.80)²+(0.60)²] = √1.74 ≈ 1.32
[75][Normal] d(A,B) ≈ 0.10  <  d(A,E) ≈ 0.39  <  d(A,D) ≈ 1.14  <  d(A,C) ≈ 1.32
[76][Normal] 가중치 (w_에너지=1.5, w_밝기=2.0, w_어쿠스틱=1.0, w_댄서빌리티=0.8)를 적용하면 D의 거리가 더 멀어진다.
[77][Normal] d_w(A,E) = √[1.5×0.01+2.0×0.09+1.0×0.01+0.8×0.04] = √0.237 ≈ 0.49
[78][Normal] d_w(A,D) = √[1.5×0.5625+2.0×0.01+1.0×0.7225+0.8×0.01] = √1.595 ≈ 1.26
[79][Normal] 세 방식을 비교하면 이렇다.
[81][Normal] 거리 밴드를 [0.30, 0.60]으로 잡으면 B(0.10), D(1.26), C(1.32)는 탈락하고 E(0.49)만 남는다. 수식으로 세렌디피티를 만들어낸 것이다.
[82][Normal] 8. 마무리
[83][Normal] 이번 탐구를 하면서 고등학교에서 배우는 미분계수가 실제로 유튜브나 애플 뮤직 같은 서비스 안에서 매일 작동하고 있다는 게 신기했다. f'(a) = lim[h→0] [f(a+h)-f(a)]/h 이 공식이 추천 알고리즘의 학습 과정을 근본에서 지탱하고 있다는 점이 인상 깊었다.
[84][Normal] 경사하강법의 흐름을 정리하면 이렇다. 손실 함수를 설계하고, 그 함수의 기울기를 미분으로 구한 다음, 기울기 반대 방향으로 가중치를 조금씩 조정한다. 이걸 반복하면 손실이 최소화되는 지점에 도달한다. 단순하지만 강력한 방법이다.
[85][Normal] 한계도 있다. 가중치 초기값을 어떻게 잡느냐가 여전히 경험에 의존한다는 점, 콘텐츠 기반 방식만으로는 사회적 취향을 반영하기 어렵다는 점, 가사나 언어처럼 정량화하기 어려운 요소는 빠져 있다는 점이다. 하지만 이런 한계 자체가 앞으로 더 공부해보고 싶은 방향을 알려줬다는 점에서 의미가 있었다.
